# Настройка Team Memory для участника команды

Эта инструкция поможет быстро подключиться к командной памяти проекта Moorinet 2.0.

---

## Шаг 1: Клонируйте репозиторий (если ещё нет)

```bash
git clone <repo-url>
cd "Moorinet 2.0"
```

---

## Шаг 2: Установите и соберите MCP сервер

```bash
cd team-memory-mcp
npm install
npm run build
```

---

## Шаг 3: Настройте Claude Code

### Вариант A: Глобальная настройка (рекомендуется)

Откройте файл `~/.claude/settings.json` и добавьте:

**Windows:**
```json
{
  "mcpServers": {
    "team-memory": {
      "command": "node",
      "args": ["D:/team-memory-mcp/dist/index.js"]
    }
  }
}
```

**macOS/Linux:**
```json
{
  "mcpServers": {
    "team-memory": {
      "command": "node",
      "args": ["/path/to/team-memory-mcp/dist/index.js"]
    }
  }
}
```

### Вариант B: Локальная настройка проекта

Создайте файл `.claude/settings.local.json` в корне проекта:

```json
{
  "mcpServers": {
    "team-memory": {
      "command": "node",
      "args": ["./team-memory-mcp/dist/index.js"]
    }
  }
}
```

---

## Шаг 4: Перезапустите Claude Code

- **VSCode**: Перезапустите окно (Ctrl+Shift+P -> "Reload Window")
- **CLI**: Закройте и откройте терминал заново

---

## Шаг 5: (Опционально) Запуск Web UI Dashboard

⚠️ **Важно:** MCP сервер и Web UI — это **два разных процесса**.

Когда вы настроили Claude Code (шаги 1-4), MCP сервер запускается автоматически в фоне при обращении к командной памяти. Но **Web Dashboard** нужно запускать отдельно.

```bash
# Откройте отдельный терминал
npm run start:web
```

После этого откройте в браузере: `http://localhost:3846`

**Web Dashboard полезен для:**
- Визуального просмотра всех записей
- Поиска и фильтрации
- Создания бэкапов через UI
- Мониторинга активности агентов

---

## Шаг 6: Проверьте подключение Claude Code

Напишите Claude:

```
Покажи текущие задачи из командной памяти
```

Или:

```
Прочитай архитектурные решения проекта
```

Если всё настроено правильно, агент использует инструмент `memory_read` и покажет записи.

**Примечание:** Web Dashboard (шаг 5) — опциональный. Claude Code будет работать с памятью и без него.

## Основные команды для работы с памятью

### Чтение

```
- Покажи все активные задачи
- Какие архитектурные решения приняты?
- Найди записи про виджеты
- Покажи известные проблемы
```

### Запись

```
- Запиши решение: выбрали shadcn/ui для UI компонентов
- Добавь задачу: реализовать drag-and-drop для виджетов
- Отметь проблему: API ещё не готов
```

### Обновление

```
- Обнови задачу XYZ - статус "выполнено"
- Измени приоритет задачи ABC на высокий
```

### Удаление

```
- Архивируй задачу XYZ
- Удали запись ABC
- Разархивируй запись DEF
```

## Web Dashboard

⚠️ **Требует отдельного запуска!**

```bash
# В отдельном терминале
npm run start:web
```

После запуска откройте: `http://localhost:3846`

Там можно:
- Просматривать все записи
- Фильтровать по категориям
- Искать по тексту
- Создавать бэкапы

## Категории памяти

| Команда | Категория | Для чего |
|---------|-----------|----------|
| "архитектура" | `architecture` | Стек, структура, паттерны |
| "задачи" | `tasks` | Текущая работа |
| "решения" | `decisions` | Принятые решения |
| "проблемы" | `issues` | Баги, блокеры |
| "прогресс" | `progress` | Завершённые этапы |

## Troubleshooting

### "MCP сервер не найден"

1. Проверьте путь в `settings.json`
2. Выполните `npm run build` в папке `team-memory-mcp`
3. Убедитесь что Node.js >= 20.0.0

### "Записи не сохраняются"

1. Проверьте права на папку `team-memory-mcp/data/`
2. Убедитесь что `memory.json` не открыт в другой программе

### "Не вижу записи других участников"

Убедитесь что все используют один и тот же файл `memory.json`. При работе локально каждый имеет свою копию.
